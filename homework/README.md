这是我在csdn上查到的模拟退火算法，解决经典的旅行商(TSP)问题

学习过程：我先查找了一些启发式算法，发现这个模拟退火算法的热度比较高，他的目的是解决寻找全局的最优解，而不是局部（这主要是和贪心算法对比），通过允许往结果要求的反方向运行（可以说是存在柳暗花明又一村吧），根据十分庞大的重复次数，从而达到确定出最优的方法。

最开始我让ai给我讲模拟退火算法，跟csdn上讲的差不多，后来他给了我一个对初学者来讲的例子：求函数最小值f(x) = x^2 + 10sinx，它存在多个局部最小值，用邻域操作去尝试，能够找到最优解，对于贪心来讲就不行

对于其中几个关键设置的理解：初始温度T0,相当于开头很大，会有很多邻居解去尝试，迭代次数L,在这个温度下搜索的次数，保证不能过大或者过小，搜索充分；降温系数a,每次迭代后下降的倍数，相当于减小分母，降低尝试概率；终止温度T\_min.

运行文件时的感觉：我找的这个作者的程序一共要查上百万次的概率，运行的时候基本上到了三分之一就已经找到正确答案了（存在一部分的缘故是根本就不可能达到设定的最小温度），能够感受到这个算法的逻辑。

这是我第一次代码接触线性回归，包含三个递进的实验：

（1）一元线性回归：只有1个特征x，预测y。核心公式b = Σ(x-x̄)(y-ȳ)/Σ(x-x̄)², a = ȳ-bx̄。这是高中数学就学过的最小二乘法，代码里用numpyj计算，用matplotlib画图。这是比较简单的，同时我也看到了一些基础的库函数的调用。我通过这个先对回归方程有认识。

（2）多元线性回归：特征扩展到8个（经度、纬度、房龄、房间数、卧室数、人口、住户数、收入），预测加州房价。前面导入库和调用库的部分是我基本不懂的，b站和csdn上的一些解释到是可以，其中也用到了测试集与训练集的分类，关键公式在于矩阵版θ = (XᵀX)⁻¹Xᵀy，fit用这个一步到位算出所有系数（ai教的），predict就是X@coef+intercept。数据要从OpenML拉取（因为sklearn.fetch\_california\_housing国内被墙了403），这一部分也是ai交的，再用StandardScaler做标准化（消除8个特征量纲不同的问题，形成标准正态分布），下面的画图都是ai弄的：最后用MSE/RMSE/R²评估——R²=0.61表示模型解释了61%的房价波动。对比sklearn的LinearRegression验证结果一致。

（3）带正则化的回归：在普通线性回归基础上加入L2惩罚项（λI），公式变成θ = (XᵀX+λI)⁻¹Xᵀy。目的是收缩系数防止过拟合。实验对比了10个λ值（0.01\~1000），λ=100发现系数更稳定。

整体感受：从一元的公式推导到多元的矩阵运算，核心逻辑其实是一样的——最小化误差平方和。sklearn封装得很方便，但自己手写一遍才能真正理解fit/predict内部在做什么。数学基础（线性代数）对理解这些算法至关重要。不过对我来讲这方面的知识确实薄弱，不过我觉得只是我对python常用的这些库的用法不会，交完作业后面会整理笔记然后学习更多的这些知识。
